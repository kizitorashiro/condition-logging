# requirements.md — グラフ視認性の改善

## 背景

コンディション分析画面のグラフ（メンタル・スキン・脳疲労の折れ線グラフ）について、
以下の理由から視認性が低い状態にある。

- 3本の線が同じ値のとき重なり、判別が困難
- X軸の日付ラベルが `YYYY-MM-DD` 形式で長く、30日表示で窮屈
- 線・点が細く、モバイルで見づらい
- 線の塗りなし＋細線の組み合わせでトレンドが追いにくい

## 変更内容

グラフの視認性を改善するため、以下の変更を行う。

### A. X軸の日付フォーマット変更
- `YYYY-MM-DD` 形式から `M/D` 形式（例: `1/21`）に短縮する
- 30日分のラベルが全て読める状態にする

### B. 線のスタイル差別化
- 色だけでなく線種でも指標を区別できるようにする
  - メンタル: 実線
  - スキン: 破線 `[6, 3]`
  - 脳疲労: 点線 `[2, 3]`
- 線が重なったときでも各指標を判別できる

### C. 線・点のサイズ拡大
- 線の太さ: デフォルト → `borderWidth: 2.5`
- 点のサイズ: デフォルト（3px）→ `pointRadius: 5`、ホバー時 `pointHoverRadius: 7`

### D. エリア塗り（透明度付き）
- 各線の下に薄い塗りつぶしを追加し、指標ごとの「領域」を視覚的に区別する
- 透明度: 約10〜15%（グラフが重くならない程度）

### E. Tooltip のカスタマイズ
- タップ/ホバーした日付の3指標値を縦に並べて同時表示する
- デフォルトのtooltipを置き換え、見やすいフォーマットに変更する

## 受け入れ条件

- [ ] X軸のラベルが `M/D` 形式で表示される
- [ ] メンタル・スキン・脳疲労の線種が異なる（実線・破線・点線）
- [ ] 線の太さ2.5、点のサイズ5で描画される
- [ ] 各線の下にエリア塗りが表示される
- [ ] Tooltipに3指標の値が同時に表示される
- [ ] 既存の機能（期間フィルター・カスタム範囲）に影響がない
- [ ] E2Eテストが通過する
