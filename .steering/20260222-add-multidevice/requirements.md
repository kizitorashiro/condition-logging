# 要求内容（複数デバイス対応）

## 概要

MVPで実現したLocalStorageによるデータ保存を、Supabaseバックエンドに移行する。
あわせてフロントエンドをGitHub Pagesでホスティングし、スマートフォン・PCの両方からアクセスできるようにする。

## 実装対象機能

### 1. 認証

- メールアドレス・パスワードによるサインインができる
- サインアウトができる
- 未サインインの場合はログイン画面のみ表示し、メインのアプリ画面は表示しない
- サインイン状態はブラウザを閉じても保持される（Supabaseのセッション管理による）
- ログイン画面にはメールアドレス入力欄・パスワード入力欄・「サインイン」ボタンのみ表示する（サインアップフォームは表示しない）
- 新規アカウント作成はSupabaseダッシュボードから手動で行う運用とし、アプリからのサインアップは無効化する
- サインイン失敗時はフォームの下にインラインでエラーメッセージを表示する（例：「メールアドレスまたはパスワードが正しくありません」）

### 2. データ保存先の移行

- コンディションログのCRUD（作成・読み取り・更新・削除）をSupabaseのDBで行う
- LocalStorageへの保存は廃止する
- 自分のデータにのみアクセスできる（Row Level Security）
- ネットワーク切断などの通信エラー時は画面上部にエラーバナーを表示する（既存のエラーバナーと同じ仕組み・5秒自動消去）

### 3. GitHub Pagesへのデプロイ

- `main` ブランチへのpushを契機にGitHub Actionsが自動ビルド・デプロイする
- デプロイ先URL: `https://kizitorashiro.github.io/condition-logging/`
- SupabaseのURLとAnon Keyは GitHub Secrets で管理し、ビルド時に環境変数として注入する

## 受け入れ条件

- [ ] ログイン画面でサインインができる
- [ ] サインイン後、記録・履歴・グラフの全機能が使える
- [ ] 記録したデータがSupabaseのDBに保存されている
- [ ] 異なるデバイスのブラウザからサインインすると同じデータが見える
- [ ] サインアウトするとログイン画面に戻る
- [ ] `main` へのpushで自動的にGitHub Pagesにデプロイされる
- [ ] デプロイされたURLにアクセスしてアプリが動作する

## 制約事項

- 認証にはSupabase Authを使用する（独自認証サーバーは構築しない）
- GitHub PagesはサブパスでホストされるためViteの `base` を `/condition-logging/` に設定する（URLルーティングは使用していないため404問題は発生しない）
- ローカル開発用の接続情報は `.env.local` に記載し、`.gitignore` に追加してリポジトリには含めない
- フロントエンドの実装言語はTypeScriptを維持する
- Supabase Anon Keyはフロントエンドに含まれるが、RLSで保護するため問題ない
- 複数ユーザーの利用は想定しない（個人利用）
- Supabaseダッシュボードでサインアップを無効化し、アカウントは手動で1件のみ作成する運用とする
