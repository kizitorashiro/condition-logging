# 要求内容（初回実装）

## 概要

`docs/product-requirements.md` に定義されたMVPスコープを実装する。
バックエンドなし・ブラウザのローカルストレージのみで動作するWebアプリケーションを構築する。

## 実装対象機能

### 1. コンディションロギング（記録画面）

- メンタル・スキン・脳疲労を1〜5の5段階で評価できる
- 各メトリクスの選択中レベル定義ラベルをボタン下に表示する
- 勤務状態を「在宅」「出社」「出張」「休み」の4択から選択できる
- 自由テキストのメモを入力できる（任意）
- 記録日は当日が自動設定され、日付ピッカーで変更できる
- 選択日に既存の記録がある場合、記録済みバッジと上書き注記を表示し、既存値をフォームに初期表示する
- 日付ピッカーで日付を変更した際も、変更先の日付に既存記録があれば、その値をフォームに読み込み直し、記録済みバッジと上書き注記を表示する
- 必須項目（メンタル・スキン・脳疲労・勤務状態）が未選択の場合、エラーメッセージを表示する
- 保存成功後、フォームを初期化する。ただし当日の記録が存在する場合はその値を読み込み直し、記録済みバッジと上書き注記を表示した状態にする
- 1日1件とし、同日に再保存した場合は上書きする

### 2. 履歴一覧（履歴画面）

- 過去のコンディションログを新しい日付順に全件一覧表示する
- 各記録にメンタル・スキン・脳疲労の数値と勤務状態を表示する
- メモがある場合のみメモを表示する
- 編集ボタンで編集モーダルを開き、日付以外の項目を編集・保存できる
- 削除ボタンで確認ダイアログを表示し、確認後に削除する

### 3. コンディション分析（グラフ画面）

- メンタル・スキン・脳疲労の3本の折れ線グラフを重ねて表示する
- X軸は日付、Y軸はレベル（1〜5）
- 記録のない日は折れ線を途切れさせる
- 期間フィルター（直近7日・30日・90日・カスタム）で表示範囲を切り替えられる（直近N日は今日を含む過去N日間）
- デフォルト表示は直近30日
- 選択した期間内に記録が1件もない場合（全体が0件の場合を含む）は、グラフを非表示にして「記録がありません」を表示する

### 4. 共通

- ボトムナビゲーションで3画面を切り替えられる
- LocalStorageにデータを保存・取得・削除できる
- LocalStorage操作失敗時はエラーメッセージを画面に表示する

## 受け入れ条件

- [ ] 記録画面でメンタル・スキン・脳疲労・勤務状態を選択し、保存できる
- [ ] 保存したデータが履歴画面に表示される
- [ ] 保存したデータがグラフ画面に折れ線として表示される
- [ ] 同日に再保存すると既存データが上書きされる
- [ ] 必須項目未選択で保存しようとするとエラーメッセージが表示される
- [ ] 履歴画面で記録を編集でき、変更が一覧とグラフに反映される
- [ ] 履歴画面で記録を削除でき、一覧とグラフから消える
- [ ] 期間フィルター（7日・30日・90日・カスタム）でグラフの表示範囲が切り替わる
- [ ] ブラウザを閉じて再度開いてもデータが保持されている
- [ ] E2Eテストが全シナリオでパスする

## E2Eテスト

ブラウザを操作して以下のシナリオを自動テストで確認する。
各ステップで画面のスクリーンショットを撮影しエビデンスとして保存する。グラフの中身は目視で確認する。

### テストシナリオ 1：記録からグラフ反映の確認

1. 記録画面を開く → スクリーンショット
2. メンタル・スキン・脳疲労・勤務状態を選択し、メモを入力する → スクリーンショット
3. 保存ボタンをクリックする → スクリーンショット
   - **アサート：** 記録済みバッジ「✓ 本日の記録は完了しています」が表示されていること
4. グラフ画面に切り替える → スクリーンショット
   - **アサート：** `<canvas>` 要素が表示されていること

### テストシナリオ 2：履歴編集の反映確認

1. 記録画面でメンタル=3・スキン=4・脳疲労=3・勤務状態=在宅を保存する → スクリーンショット
2. 履歴画面に切り替える → スクリーンショット
   - **アサート：** 履歴カードにメンタル=3・スキン=4・脳疲労=3 が表示されていること
3. 編集ボタンをクリックする → スクリーンショット
   - **アサート：** 編集モーダルが開いていること
4. メンタルを3から5に変更して保存する → スクリーンショット
   - **アサート：** モーダルが閉じていること
5. 履歴一覧を確認する → スクリーンショット
   - **アサート：** 履歴カードにメンタル=5 が表示されていること
6. グラフ画面に切り替える → スクリーンショット
   - **アサート：** `<canvas>` 要素が表示されていること

### 事前処理

各テストシナリオの実行前に `localStorage.clear()` を呼び出し、データが空の状態から開始する。

### スクリーンショットの保存先

`e2e/screenshots/` ディレクトリに保存する。ファイル名は `[シナリオ番号]-[ステップ番号]-[説明].png` とする。
例: `01-03-recorded-badge.png`

## 制約事項

- バックエンドは使用しない（LocalStorageのみ）
- E2EテストにはPlaywrightを使用する
- ユニットテスト・統合テストは書かない
- 使用ライブラリはChart.js・Pico CSS（Class-conditional版）・Interフォント（Google Fonts）のみ
- スマートフォン・PCの両方のブラウザで動作すること（MVPはローカルファイルで確認）
