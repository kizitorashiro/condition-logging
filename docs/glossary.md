# ユビキタス言語定義

開発・設計・会話で使用する用語を統一する。コード・ドキュメント・UIいずれにおいても、ここで定義した用語を一貫して使用する。

---

## プロジェクト用語

| 用語 | 定義 |
|------|------|
| MVP（Minimum Viable Product） | 最小限の機能で動作する最初のバージョン。本プロジェクトでは「ロギング・履歴・グラフ表示をローカルストレージで動作させること」をMVPの完了とする |

---

## ドメイン用語

| 日本語 | 英語（コード上） | 定義 |
|--------|---------------|------|
| コンディション | Condition | メンタル・スキン・脳疲労・勤務状態を総合した、その日の心身の状態 |
| コンディションログ | ConditionLog | 1日分のコンディションを記録したデータ1件 |
| ロギング | Logging | コンディションを記録する行為の概念。コード上では `save()` メソッドなどで表現され、`Logging` という名のクラスや関数は存在しない |
| 分析 | Analysis | 記録されたコンディションログを時系列グラフで可視化し、傾向を把握する機能の総称 |
| メンタル | Mental | 気分・感情の安定度を表すメトリクス。1〜5の5段階で評価する |
| スキン | Skin | 肌の状態（荒れ・くすみなど外見への影響）を表すメトリクス。1〜5の5段階で評価する |
| 脳疲労 | BrainFatigue | 集中力・思考力の低下度を表すメトリクス。1〜5の5段階で評価する |
| メトリクス | Metrics | メンタル・スキン・脳疲労の総称。総称する場合は `Metrics`（複数）、個々を指す場合は `Mental`・`Skin`・`BrainFatigue` と呼ぶ。コード上のフィールド名は camelCase で `mental`・`skin`・`brainFatigue` |
| レベル | Level | メトリクスの評価値（1〜5の整数） |
| 勤務状態 | WorkStyle | その日の働き方を表す4択の値 |
| メモ | Memo | コンディションに関する自由記述の補足情報（任意） |
| 記録日 | LogDate | コンディションログが記録された日付。コード上は `string` 型のフィールド `logDate`（YYYY-MM-DD形式）として扱う。JavaScriptの `Date` 型とは別物 |
| 最終更新日時 | UpdatedAt | コンディションログが最後に保存・上書きされた日時。ISO 8601形式の文字列。システムが自動付与しユーザーは変更できない |
| 上書き | Overwrite | 同じ記録日のコンディションログを新しい内容で置き換えること |

---

## 勤務状態（WorkStyle）の列挙値

| 日本語（UI表示） | 英語（コード上） | 定義 |
|---------------|---------------|------|
| 在宅 | `remote` | 自宅でリモートワークをした日 |
| 出社 | `office` | オフィスに出勤した日 |
| 出張 | `business_trip` | 出張で勤務した日 |
| 休み | `day_off` | 休日・有給・その他で勤務しなかった日 |

---

## メトリクスのレベル定義

### メンタル

| レベル | UI表示ラベル |
|-------|------------|
| 1 | 焦燥感、もう無理感あり |
| 2 | 予兆あり |
| 3 | 安定 |
| 4 | 前向き |
| 5 | 躁傾向 |

### スキン

| レベル | UI表示ラベル |
|-------|------------|
| 1 | 全身不調 |
| 2 | 部分的に不調 |
| 3 | 回復傾向 |
| 4 | 一部出てるが気にならない |
| 5 | 出てない |

### 脳疲労

| レベル | UI表示ラベル |
|-------|------------|
| 1 | 限界 |
| 2 | 集中力なし |
| 3 | 集中力ダウン |
| 4 | 余力あり |
| 5 | 余裕 |

## UI / UX用語

| 日本語 | 英語（コード・コメント上） | 定義 |
|--------|----------------------|------|
| 記録画面 | Log Screen | コンディションを新規入力・保存する画面 |
| 履歴画面 | History Screen | 過去のコンディションログを一覧表示する画面 |
| グラフ画面 | Chart Screen | コンディションの時系列推移を折れ線グラフで表示する画面 |
| ボトムナビゲーション | Bottom Navigation | 画面下部に固定された3画面切り替えタブ |
| 編集モーダル | Edit Modal | 履歴画面から過去のログを編集するためのモーダルダイアログ |
| 5段階評価ボタン | Rating Button | メトリクスのレベルを選択する1〜5の番号ボタン |
| セグメントボタン | Segment Button | 勤務状態を選択する4択のボタングループ |
| 記録済みバッジ | Recorded Badge | 記録画面で選択日のログが既に存在することを示すバッジ |
| 上書き注記 | Overwrite Notice | 既存ログがある日に保存しようとした際に表示される警告テキスト |

---

## コード上のクラス・インターフェース名

| 名称 | 種別 | 役割 |
|------|------|------|
| `ConditionLog` | type（object） | コンディションログ1件のデータ構造 |
| `WorkStyle` | type（union） | 勤務状態の列挙型 |
| `ConditionLogRepository` | interface | データアクセスの抽象インターフェース |
| `LocalStorageConditionLogRepository` | class | LocalStorageを使ったリポジトリのMVP実装 |
| `ApiConditionLogRepository` | class | バックエンドAPIを使ったリポジトリの将来実装 |
| `ConditionLogService` | class | バリデーション・ビジネスロジックを担うサービス層 |
